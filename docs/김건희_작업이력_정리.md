이 문서는 GitHub PR 제목만으로 작업자를 판단하지 않고, `main`의 non-merge 커밋 작성자, PR merge의 두 부모 사이 커밋 작성자, 파일 최초 실질 구현 커밋, 현재 파일의 `git blame`을 함께 확인해 정리한 개인 작업 문서다.

저장소 소유자와 브랜치 경로가 `rlarjsgml1/...`로 표시되어도 그 PR의 실제 커밋 작성자가 다른 팀원이면 김건희 작업으로 계산하지 않았다. 초기 프로젝트 설정에서 만들어진 짧은 placeholder 파일도 이후 다른 팀원이 실질 구현했다면 개인 구현 화면으로 계산하지 않았다.

## 1. 기여 분류 기준

| 분류 | 기준 |
|---|---|
| 주도 구현 | 최초 실질 구현과 현재 핵심 코드의 대부분이 김건희 커밋으로 확인되는 작업 |
| 공동 구현 | 김건희가 초기 또는 주요 부분을 구현했지만 다른 작업자가 핵심 기능을 크게 확장한 작업 |
| 후속 정합화 | 다른 작업자가 주 구현한 화면·API에서 계약, 오류 처리, UX, 접근성, 데이터 표시를 보완한 작업 |
| 문서·검증 | 화면 정의서, API gap, DB 실행 가이드, 디자인 제안, 코드·백엔드 대조 작업 |
| 제외 | 다른 작업자 주 구현이며 김건희 기여가 없거나 매우 작은 작업 |

화면이나 API 파일이 현재 저장소에 존재한다는 사실만으로 개인 작업으로 분류하지 않는다.

## 3. 역할 요약

React·TypeScript 기반 Front 프로젝트의 초기 기반, 인증 API 연동, 마이페이지 핵심 화면, 대여 현황, 출석·포인트·통계·알림, 도서관 위치 검색과 네이버 지도, 일부 관리자 운영 화면을 주도했다.

또한 공통 Modal·Table·Pagination, Admin 레이아웃, 프로필 공유 Context, 라우터 코드 스플리팅, 디자인 토큰과 접근성 규칙을 구현했다. 다른 팀원이 만든 도서·검색·상세, 결제, 전자책 EPUB 렌더러, 일부 Admin 화면에서는 API 계약과 오류 처리, UX를 후속 보완했다.

## 14. 발표용 핵심 성과

1. 마이페이지, 출석, 포인트·쿠폰, 통계, 알림을 실제 API 흐름과 연결했다.
2. 대여 현황과 User/Admin 도서관 기능을 직접 구현했다.
3. Naver Maps를 이용해 도서관 결과를 지도와 목록으로 양방향 연동했다.
4. 관리자 대여·결제·공지·통계·감사로그·도서관 관리 화면을 주도했다.
5. Modal·Table·Pagination과 Admin 레이아웃을 공통화했다.
6. API 실패를 가짜 데이터·가짜 성공으로 숨기던 문제를 제거했다.
7. 백엔드 DTO와 상태·날짜·지역 계약을 실제 코드와 대조해 정합화했다.
8. 타 작업자 주도 화면은 공동·후속 기여로 분리해 개인 성과를 과장하지 않았다.

## 15. 한 문장 소개

React·TypeScript 기반 전자책 플랫폼에서 마이페이지·대여 현황·출석·알림·통계·도서관 지도와 일부 관리자 운영 화면을 주도하고, 공통 UI 체계 및 실제 백엔드 API 계약 정합화를 담당했으며, 전자책 뷰어·결제·도서·검색·일부 Admin 화면은 팀 구현 위에 오류 처리와 UX 개선으로 기여했다.
