## 2. 핵심 담당 기능

### 2.1 Front 프로젝트 기반 구축

- React + Vite + TypeScript 프로젝트 초기 생성 및 폴더 구조 구성
- React Router 기반 User/Admin/MyPage 라우팅 구성
- Axios 공통 클라이언트와 인증 토큰 헤더 처리
- 회원 전용 마이페이지·대여·뷰어와 관리자 페이지 접근을 위한 `ProtectedRoute` 적용
- 페이지 컴포넌트 `React.lazy` 전환으로 초기 번들 코드 스플리팅
- TypeScript language server와 Biome 개발 도구 도입
- 로컬 DB 연결 및 실행·트러블슈팅 문서 작성

### 2.2 인증·회원·마이페이지

- 회원가입·로그인 API 연결 및 로그인 상태를 헤더와 동기화
- 회원가입 실시간 입력 검증, 비밀번호 확인, 약관 동의 모달 구현
- 검증 실패 시 오류 입력칸 자동 스크롤·포커스 이동
- 아이디 찾기와 비밀번호 재설정 화면 및 로그인 딥링크 구현
- 마이페이지 프로필, 회원정보 수정, 회원 탈퇴 API 연결
- 프로필 데이터를 `MyProfileContext`로 공유하여 중복 호출 감소
- 백엔드 규칙에 맞춘 성별 값과 회원 등급·포인트 표시 정합화

### 2.3 대여·결제·전자책 열람

- 내 대여 현황 화면과 상태별 목록 표시
- 무료/유료 대여 및 결제 흐름 화면 정리
- 전자책 뷰어 구현
  - EPUB 로딩
  - 이전·다음 페이지 이동
  - 읽기 진행률 저장
  - 북마크 조회·등록·삭제
  - 페이지 전환 페이드 효과
- `epubjs` 실행 시 `require is not defined` 오류 해결
- 결제 내역과 결제 완료 화면의 필드·레이아웃 정리

### 2.4 사용자 활동 기능

- 출석체크 화면 제작 및 실제 API 연결
- 포인트·쿠폰 목록과 사용 이력 화면 구현
- 독서 통계 화면과 차트 시각화 구현
- 알림 목록·읽음 처리·유형 필터 화면 구현
- 헤더 알림 벨과 읽지 않은 알림 개수 배지 구현
- 헤더 도서 검색 자동완성에 디바운스, 요청 취소, 오래된 응답 방지 처리

- 희망도서 신청 및 신청 내역 화면 구현
- 최근 읽은 책, 즐겨찾기, 내 신고 내역 화면 구현

### 2.5 도서 탐색·상세·도서관 위치

- 도서 목록·검색·상세 API 연결 및 오류 처리
- 도서 상세 리뷰 작성·수정·삭제 API 연결

### 5.3 API 정합화 원칙

- API 실패를 임시 데이터나 로컬 성공으로 숨기지 않는 것을 원칙으로 정리했으며, U-001·U-026·A-001·A-013·A-016·A-017의 잔존 fallback은 9절에 예외로 명시
- 백엔드 DTO가 변경되면 Front 타입 또는 API 어댑터에서 필드 정합화
- 서버 페이지네이션과 클라이언트 필터의 범위를 구분
- 허용된 상태 전이와 요청 조건을 UI 단계에서도 제한
- 날짜는 백엔드 `LocalDateTime` 형식에 맞춰 직렬화
- API가 없는 기능은 실제 저장처럼 보이지 않도록 문서와 UI 상태를 분리

## 6. 주요 문제 해결 사례

### 6.1 가짜 데이터·가짜 성공 제거

- 도서 주 목록·검색 결과의 핵심 실패 경로에서 샘플 데이터로 정상처럼 보이던 fallback 제거
- 관리자 대여·결제·도서관 목록 실패 시 가짜 데이터 표시 제거
- 관리자 신고 처리에서 API가 실패해도 성공처럼 목록을 갱신하던 동작 제거
- API 오류를 사용자와 관리자가 확인할 수 있도록 오류 상태 정리
- 홈·일부 카테고리와 관리자 대시보드·통계·감사 로그·쿠폰 화면에는 샘플 fallback이 남아 있어 운영 완료 항목과 구분

### 6.2 백엔드 계약 불일치 해결

- 회원정보 수정의 성별 값과 요청 필드를 백엔드 규칙에 맞춤
- 마이페이지 등급·포인트 응답 필드 정합화
- 추천 도서 응답 envelope 파싱 오류 수정
- 독서 통계 API 응답 매핑과 차트 라벨 정정
- 도서관 검색의 지역·구군 파라미터와 응답 구조 정리
- 관리자 신고 처리의 허용 상태 변경, 리뷰 제재 조건, 날짜 형식 보완

### 6.3 UX·접근성 개선

- 회원가입 검증 실패 시 해당 필드로 자동 이동
- 키보드 사용자를 위한 전역 `:focus-visible` 표시
- 버튼 active 피드백과 카드 기본 상태 그림자 추가
- 모달·테이블·페이지네이션을 공통 컴포넌트로 통일
- 반응형 브레이크포인트와 페이지 최대 너비 체계 통일
- 애니메이션 감소 설정 사용자를 위한 `prefers-reduced-motion` 대응

### 6.4 성능·유지보수 개선

- 라우터 페이지를 lazy loading으로 전환
- 마이페이지 프로필 Context 도입으로 중복 요청 감소
- 공통 컴포넌트로 반복 마크업과 스타일 축소
- CSS 색상 하드코딩을 브랜드 토큰으로 교체
- 일부 화면에 API·역할을 설명하는 모듈 개요 주석 추가
