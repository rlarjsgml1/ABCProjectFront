# ABC 프로젝트 User Front 분업표

## 1. 분업 기준

- Admin 화면은 이번 분업에서 제외한다.
- User Front 화면만 분배한다.
- 건희는 이미 Admin 화면을 담당했으므로 User Front 화면을 과도하게 배정하지 않는다.
- 진희누나는 토요일부터 빠지기 때문에 비교적 빠르게 마무리할 수 있는 인증 화면을 담당한다.
- `U-011 전자책 뷰어`는 구현 난이도와 범위를 내일 다시 논의하기 위해 담당자 미정으로 보류한다.

---

## 2. 담당자별 분업

### 진희누나

| 화면 ID | 화면명 |
|---|---|
| `U-002` | 회원가입 |
| `U-003` | 로그인 |
| `U-004` | 아이디 찾기 |

#### 담당 범위

- 화면 UI 구현
- 입력값 검증
- 회원가입 API 연결
- 로그인 API 연결
- 아이디 찾기 API 연결
- 오류 메시지 표시
- 로그인 성공 후 기본 화면 이동 처리

| 화면 ID | 화면명 |
|---|---|
| `U-010` | 내 대여 현황 |
| `U-014` | 마이페이지 |

- Admin 화면을 이미 담당했으므로 추가 User 화면은 가급적 배정하지 않는다.

## 3. 담당자 미정 화면

| 화면 ID | 화면명 | 상태 |
|---|---|---|
| `U-011` | 전자책 뷰어 | 담당자 미정 / 내일 논의 |

### 보류 이유

전자책 뷰어는 다음 기능이 포함되어 있어 다른 화면보다 구현 난도가 높다.

- 이전/다음 페이지 이동
- 특정 페이지 이동
- 현재 페이지와 전체 페이지 표시
- 읽기 진행률 저장
- 북마크 추가 및 삭제
- 북마크 목록
- 처음으로 가기
- 자동 페이지 넘김
- 뷰어 종료 시 진행 상태 저장

따라서 팀원별 현재 작업량과 백엔드 API 구현 상태를 확인한 후 담당자를 결정한다.

---

## 4. 공통 인증 작업

진희누나는 인증 화면과 기본 API 연결을 담당하고, 프로젝트 전체에 영향을 주는 공통 인증 처리는 건희가 정리한다.

### 진희누나

- 회원가입 폼 및 API
- 로그인 폼 및 API
- 아이디 찾기 폼 및 API
- 입력 오류 및 로그인 실패 메시지
- 로그인 성공 결과 전달

### 건희

- `accessToken` 저장
- Axios Request Interceptor
- `Authorization: Bearer {accessToken}` 적용
- USER / ADMIN 권한별 이동
- 새로고침 후 로그인 상태 복구
- 로그아웃 시 토큰 제거
- `401 Unauthorized` 공통 처리
